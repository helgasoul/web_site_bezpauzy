# Решение проблемы с кодировкой пути

## Проблема

Терминал не может правильно обработать путь с кириллицей и специальными символами:
```
/Users/olgapuchkova/Documents/Документы — MacBook Air — Olga/...
```

## Решения

### ✅ Решение 1: Использовать терминал Cursor (самое простое)

1. **Откройте терминал в Cursor:**
   - Нажмите `Ctrl + `` (обратная кавычка) или
   - Меню: `Terminal` → `New Terminal`

2. **Cursor автоматически откроет терминал в правильной директории проекта**

3. **Выполните:**
   ```bash
   npm install
   ```

### ✅ Решение 2: Через Finder (macOS)

1. Откройте **Finder**
2. Перейдите в папку проекта (где находится `package.json`)
3. **Перетащите папку** в окно **Terminal.app**
4. Выполните:
   ```bash
   npm install
   ```

### ✅ Решение 3: Создать алиас (для постоянного использования)

Добавьте в `~/.zshrc` или `~/.bash_profile`:

```bash
alias bezpauzy='cd "$HOME/Documents/Документы — MacBook Air — Olga/PERSONAL/МОИ ПРОЕКТЫ/ПЛАТФОРМА ЖЕНСКОГО ЗДОРОВЬЯ/Проект Без | паузы/Бот Без|паузы/САЙТ"'
```

Затем:
```bash
source ~/.zshrc  # или ~/.bash_profile
bezpauzy
npm install
```

### ✅ Решение 4: Использовать относительный путь

Если вы уже в директории `Documents`:

```bash
cd "Документы — MacBook Air — Olga/PERSONAL/МОИ ПРОЕКТЫ/ПЛАТФОРМА ЖЕНСКОГО ЗДОРОВЬЯ/Проект Без | паузы/Бот Без|паузы/САЙТ"
npm install
```

### ✅ Решение 5: Создать симлинк (символическую ссылку)

```bash
# Создать симлинк в домашней директории
ln -s "$HOME/Documents/Документы — MacBook Air — Olga/PERSONAL/МОИ ПРОЕКТЫ/ПЛАТФОРМА ЖЕНСКОГО ЗДОРОВЬЯ/Проект Без | паузы/Бот Без|паузы/САЙТ" ~/bezpauzy-site

# Использовать симлинк
cd ~/bezpauzy-site
npm install
```

## Почему это происходит?

- Терминал использует UTF-8, но некоторые оболочки могут неправильно обрабатывать кириллицу
- Специальные символы (тире, пробелы) требуют экранирования
- Разные версии shell (bash, zsh) могут по-разному обрабатывать пути

## Рекомендация

**Используйте терминал Cursor** — он автоматически открывается в правильной директории проекта и правильно обрабатывает пути с кириллицей.

## Проверка

После установки проверьте:
```bash
ls node_modules/ | head -5
```

Если видите папки (`react`, `next`, и т.д.) — всё установлено правильно! ✅
