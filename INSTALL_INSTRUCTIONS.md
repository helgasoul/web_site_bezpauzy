# Инструкция по установке зависимостей

## Проблема

Из-за проблем с кодировкой пути в терминале, автоматическая установка может не работать.

## Решение

### Вариант 1: Через терминал Cursor (рекомендуется)

1. Откройте терминал в Cursor (Terminal → New Terminal)
2. Убедитесь, что вы находитесь в директории проекта (там, где `package.json`)
3. Выполните:
   ```bash
   npm install
   ```

### Вариант 2: Через скрипт

1. Откройте терминал в директории проекта
2. Выполните:
   ```bash
   chmod +x install-dependencies.sh
   ./install-dependencies.sh
   ```

### Вариант 3: Через Finder (macOS)

1. Откройте Finder
2. Перейдите в директорию проекта
3. Правой кнопкой на папке → "Службы" → "Новый терминал в папке" (если установлено)
4. Или перетащите папку в окно Terminal.app
5. Выполните `npm install`

## Что будет установлено

- **React** и **React DOM** (основа UI)
- **Next.js 14** (фреймворк)
- **TypeScript** (типизация)

- **Framer Motion** (анимации)

- И другие зависимости

## После установки

После успешной установки все ошибки TypeScript должны исчезнуть, и вы сможете запустить проект:

```bash
npm run dev
```

Откройте http://localhost:3000 в браузере.

## Проверка

Если установка прошла успешно, вы увидите:
- Папку `node_modules/` в директории проекта
- Файл `package-lock.json`
- Отсутствие ошибок TypeScript в IDE
