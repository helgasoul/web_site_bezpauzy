# Исправление проблемы с кириллицей в PDF

## Проблема
При генерации PDF кириллица отображается как набор непонятных символов вместо читаемого текста.

## Причина
jsPDF 3.x требует конвертации TTF шрифтов через специальный конвертер. Простое добавление TTF файлов через `addFileToVFS` и `addFont` не работает.

## Решение

### Вариант 1: Использовать конвертер шрифтов jsPDF (рекомендуется)

1. Откройте конвертер: https://github.com/parallax/jsPDF/blob/master/fontconverter/fontconverter.html
2. Загрузите шрифты:
   - DejaVuSans.ttf (normal)
   - DejaVuSans-Bold.ttf (bold)
3. Скачайте сгенерированные JavaScript файлы
4. Добавьте их в проект и импортируйте перед использованием jsPDF

### Вариант 2: Использовать готовые конвертированные шрифты

Можно использовать готовые конвертированные шрифты из npm пакетов или создать их самостоятельно.

### Вариант 3: Использовать альтернативную библиотеку

Для серверной генерации PDF с кириллицей можно использовать:
- PDFKit (Node.js)
- Puppeteer (рендеринг HTML в PDF)
- @react-pdf/renderer (уже установлен в проекте)
